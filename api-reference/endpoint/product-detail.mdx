## 产品详情概述

获取指定产品的详细信息，包括产品配置、物模型信息、设备数量统计和产品状态等完整数据。

## 产品类型详解

<Tabs>
  <Tab title="普通设备 (NORMAL)">
    **直连设备**: 直接连接到IoT平台的标准设备
    - 支持MQTT、CoAP、HTTP协议
    - 独立的设备身份认证
    - 完整的生命周期管理
    - 适用于传感器、执行器等单体设备
  </Tab>
  
  <Tab title="网关设备 (GATEWAY)">
    **网关设备**: 作为其他设备连接代理的设备
    - 管理多个子设备的连接
    - 协议转换和数据汇聚
    - 子设备拓扑关系管理
    - 适用于Zigbee网关、LoRa网关等
  </Tab>
  
  <Tab title="子设备 (CHILD)">
    **子设备**: 通过网关连接到平台的设备
    - 通过网关进行数据通信
    - 依赖网关的连接状态
    - 简化的设备管理流程
    - 适用于Zigbee节点、LoRa节点等
  </Tab>
</Tabs>

```python Python
import requests

def get_product_detail(token, product_key):
    """
    获取产品详细信息
    
    Args:
        token: JWT认证令牌
        product_key: 产品PK
        
    Returns:
        产品详细信息
    """
    url = "https://iot-api.quectelcn.com/v2/quecproductmgr/r3/openapi/product/detail"
    
    params = {
        "productKey": product_key
    }
    
    headers = {
        "Authorization": token,
        "Content-Type": "application/json"
    }
    
    response = requests.get(url, params=params, headers=headers)
    response.raise_for_status()
    
    return response.json()

# 使用示例
product_detail = get_product_detail(token, "p12345")
product = product_detail["data"]

print(f"产品名称: {product['productName']}")
print(f"产品类型: {product['productType']}")
print(f"数据格式: {product['dataFormat']}")
print(f"网络类型: {product['networkType']}")

# 根据产品类型处理不同逻辑
if product["productType"] == "GATEWAY":
    print("这是网关产品，支持子设备管理")
elif product["productType"] == "NORMAL":
    print("这是普通设备产品")
```

## 产品字段详解

<AccordionGroup>
  <Accordion title="数据格式 (dataFmt)">
    **物模型格式 (3)**: 使用标准化的TSL物模型
    - 结构化的属性、服务、事件定义
    - 支持数据类型验证和转换
    - 便于平台统一处理和展示
    
    **透传格式 (0)**: 透传/自定义数据格式
    - 透传原始数据，不进行解析
    - 需要应用层自行处理数据格式
    - 适用于特殊协议或二进制数据
  </Accordion>
  
  <Accordion title="联网方式 (netWay)">
    **WiFi(MQTT) (1)**: Wi-Fi网络连接
    - 2.4GHz/5GHz频段，使用MQTT协议
    - 局域网部署，成本较低
    - 依赖WiFi基础设施
    
    **2G/3G/4G/5G(MQTT) (2)**: 蜂窝网络连接
    - 2G/3G/4G/5G网络，使用MQTT协议
    - 广域覆盖，适合移动场景
    - 运营商计费模式
    
    **NB-IoT(LwM2M) (3)**: 窄带物联网
    - NB-IoT网络，使用LwM2M协议
    - 低功耗广域网络
    - 适合大规模IoT部署
    
    **蓝牙(HTTP) (5)**: 蓝牙连接
    - 蓝牙通信，使用HTTP协议
    - 近距离通信，低功耗特性
    - 适合可穿戴设备
  </Accordion>
  
  <Accordion title="设备类型 (accessType)">
    **直连设备 (0)**: 直连节点
    - 设备直接连接到IoT平台
    - 独立的设备身份和连接
    - 完整的设备管理功能
    
    **网关设备 (1)**: 网关节点
    - 作为其他设备的连接代理
    - 管理子设备的连接和通信
    - 支持协议转换和数据汇聚
    
    **网关子设备 (2)**: 子设备节点
    - 通过网关连接到平台
    - 依赖网关的连接状态
    - 简化的设备管理流程
  </Accordion>
  
  <Accordion title="接入平台 (connectPlatform)">
    **移远云平台 (0)**: 移远自有IoT平台
    - 完整的IoT设备管理功能
    - 支持全套API和服务
    - 推荐的接入方式
    
    **电信AEP平台 (1)**: 中国电信AEP平台
    - 基于电信AEP平台的设备接入
    - 部分功能可能受限
    - 特定场景下使用
  </Accordion>
  
  <Accordion title="产品配置 (productConfig)">
    **AI能力状态 (aiCapabilityStatus)**: AI功能开关
    - false: AI能力关闭
    - true: AI能力开启
    - 默认为false
    - 影响设备的AI相关功能
  </Accordion>
</AccordionGroup>

## 错误处理

| 错误码 | 说明 | 解决方案 |
|---------|------|----------|
| 50006 | ProductKey为空 | 提供有效的productKey参数 |
| 70017 | 产品不存在 | 验证产品是否存在且可访问 |
| 70089 | 权限受限 | 检查SaaS应用的产品访问权限 |
| 401 | Token无效或过期 | 刷新JWT token后重试 |

| 状态 | 说明 | 设备注册 |
|------|------|----------|
| DEVELOPING | 开发中 | ❌ 不可注册 |
| PUBLISHED | 已发布 | ✅ 可以注册 |
| DEPRECATED | 已废弃 | ❌ 不可注册 |
